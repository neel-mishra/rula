# PR Review Checklist

## Intent and Scope
- [ ] PR matches plan outcome
- [ ] Scope creep is documented or removed

## Core Review Focus
- [ ] Business/domain logic is correct
- [ ] UX/copy behavior is clear and non-regressive
- [ ] Risk and rollback posture is acceptable

## Technical Safety
- [ ] Contracts/schemas remain compatible
- [ ] Security-sensitive paths are validated
- [ ] Observability for changed behavior exists

## Findings Management
- [ ] P1 issues resolved
- [ ] P2 decisions documented (fix/defer + owner + due date)
- [ ] P3 queued if worth compounding

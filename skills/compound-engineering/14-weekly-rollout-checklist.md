# Weekly Rollout Checklist (One Page)

## Week 1 Objective

Run at least one real feature through Plan -> Work -> Review -> Compound with strict gates enabled.

## Monday - Setup

- [ ] Confirm single feature owner for each active feature
- [ ] Confirm strict high-risk gate policy is active
- [ ] Confirm hard-blocker classes are visible to all contributors
- [ ] Copy templates/checklists into active working docs paths

## Tuesday - Planning Quality

- [ ] Produce at least one approved plan using `04-templates/plan-template.md`
- [ ] Classify risk tier and update risk register
- [ ] Explicitly identify affected components/contracts

## Wednesday - Execution Quality

- [ ] Execute planned scope only (track intentional deviations)
- [ ] Capture validation evidence (tests/lint/type checks/manual checks)
- [ ] Record rollout and rollback strategy for high-risk work

## Thursday - Review and Triage

- [ ] Review using `04-templates/review-template.md`
- [ ] Resolve all P1 findings
- [ ] Record P2 deferrals with owner + due date
- [ ] Confirm merge readiness against `05-checklists/quality-gate-checklist.md`

## Friday - Compound and Score

- [ ] Publish compound artifact using `04-templates/compound-retrospective-template.md`
- [ ] Update at least one checklist/template/rule based on learnings
- [ ] Score maturity for critical domains in `11-system-component-breakout.md`
- [ ] Log next week’s system-improvement actions

## End-of-Week Exit Criteria

- [ ] One complete feature packet exists (plan/review/compound)
- [ ] Weekly metrics and compounding notes are recorded
- [ ] No unresolved P1 findings
- [ ] Critical domain maturity scores updated with evidence

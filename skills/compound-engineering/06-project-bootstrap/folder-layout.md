# Folder Layout Reference

## Canonical Layout

```text
docs/
  plans/
  reviews/
  solutions/
  architecture/
  runbooks/
todos/
```

## Purpose by Folder

- `docs/plans/`: approved implementation plans
- `docs/reviews/`: PR/review findings and outcomes
- `docs/solutions/`: compound artifacts with reusable learnings
- `docs/architecture/`: system boundaries and contracts
- `docs/runbooks/`: release, incident, and operational procedures
- `todos/`: triaged and prioritized follow-up work

# Project Bootstrap Starter

Use this starter to initialize Compound Engineering on a new project.

## Recommended Structure

```text
project/
  docs/
    plans/
    reviews/
    solutions/
    architecture/
    runbooks/
  todos/
  AGENTS.md (or CLAUDE.md)
```

## Initialization Sequence

1. Copy templates from `skills/compound-engineering/04-templates/`.
2. Copy checklists from `skills/compound-engineering/05-checklists/`.
3. Define project-specific hard blockers and risk tiers.
4. Define owners and approval paths.
5. Run first baseline maturity score for critical domains.
6. Schedule weekly compounding ritual.

## Minimum Startup Artifacts

- Initial `plan` document for first feature/system.
- Initial `risk register`.
- `quality gate` checklist customized for project constraints.
- Baseline `component maturity` worksheet.

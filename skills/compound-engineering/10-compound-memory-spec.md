# Compound Memory Spec

## Purpose

Define how solved problems become reusable system memory.

## Required Metadata (YAML)

- `title`
- `owner`
- `date`
- `tags`
- `component_domains`
- `risk_tier`
- `related_files`
- `related_prs`

## Required Content Sections

1. Problem pattern and trigger conditions
2. Solution pattern and rationale
3. Prevention rules and guardrails
4. Validation evidence
5. System updates (templates/checklists/rules)

## Quality Bar

A compound artifact is complete only if:
- It is findable (metadata complete)
- It is reusable (solution + prevention clearly written)
- It is verifiable (evidence included)
- It updates the system (at least one reusable asset changed)

# 30/60/90-Day Roadmap

## 0-30 Days (Foundation)

- Publish and adopt framework docs/templates/checklists.
- Enforce strict high-risk merge bar using `05-checklists/quality-gate-checklist.md`.
- Start weekly compounding review.
- Baseline maturity scoring for critical domains.

## 31-60 Days (Operationalization)

- Standardize triage flow and SLAs from `09-governance-and-triage.md`.
- Require compound artifact for all high-risk merged changes.
- Tighten release readiness and rollout plans on each release.
- Improve visibility into unresolved P2/P3 debt.

## 61-90 Days (Scale and Hardening)

- Increase parallel feature throughput with adaptive risk-based boundaries.
- Raise evidence quality for maturity claims across critical domains.
- Reduce cycle time and defect escape using weekly compounding updates.
- Lock in repeatable launch/release discipline as default operating behavior.

## Success Criteria

- High-risk work consistently meets gates without ad hoc exceptions.
- Weekly compounding produces measurable metric improvement.
- Framework is reused by new projects with minimal onboarding friction.

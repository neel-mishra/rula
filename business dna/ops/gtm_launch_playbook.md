# GTM Launch Playbook: Rula Agent-Orchestrated Motion

## 1. Phase 1: Foundation & Alignment (Weeks 1-2)
- **Goal:** Ensure launch narrative aligns with Rula business context and ICP.
- **Agent instructions:**
  1. Read `@core/business_context.md` and `@core/product_dna.md`.
  2. Select target segment from `@core/ideal_customer_profile.md`.
  3. Confirm framing against `@core/competitor_landscape.md`.

## 2. Phase 2: Asset Creation (Weeks 3-4)
- **Goal:** Generate internal and external assets with structural consistency.
- **Agent instructions:**
  1. Draft PRD from `@ops/prd.md`.
  2. Choose primary angle from `@identity/messaging_pillars.md`.
  3. Draft stakeholder brief using `@identity/style_guide_internal.md`.

## 3. Phase 3: Multi-Channel Execution (Week 5)
- **Goal:** Launch coordinated channel assets with sourced claims.
- **Agent instructions:**
  1. Build copy via `@identity/ad_copy_frameworks.md`.
  2. Build release comms via `@ops/release_announcement_workflow.md`.
  3. Build visual briefs via `@identity/creative_direction.md`.

---

## 4. Launch Day Rituals
- Confirm final claim provenance links before publishing.
- Run stakeholder sync with decision log and owner assignments.
- Capture first-wave feedback using `@ops/meeting_summarizer_protocol.md`.

## 5. Post-Launch: Feedback Loop
- Compare performance against PRD success metrics.
- Update `@ops/customer_summary_intelligence.md` with new signals.
- Refresh pillars/ICP assumptions where data shows drift.

---

### AI Agent Orchestration Rule
Treat this as sequential unless explicitly approved to parallelize:
1. Foundation alignment must complete before broad channel execution.
2. If required context file is missing or stale, stop and flag immediately.
3. If claim confidence is low, downgrade to review-required mode.

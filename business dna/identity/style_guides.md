# Style Guides: Rula

## 1. Customer-Facing Editorial (Default)

- **Audience:** People seeking therapy/psychiatry, often navigating stress and insurance complexity.
- **Goal:** Build trust and reduce friction from first touch to care start.

### Voice
| Principle | Do | Don't |
| :--- | :--- | :--- |
| **Clear** | Explain next step plainly | Hide process behind vague language |
| **Supportive** | Validate common barriers | Use shame or urgency pressure |
| **Evidence-aware** | Use sourced claims | Use unsourced superlatives |
| **Human** | Write like a helpful guide | Sound robotic or overly clinical |

### Key phrases (examples)
- "in-network mental healthcare"
- "find a provider that fits your needs"
- "coverage verified before booking"
- "therapy and psychiatry options"

---

## 2. Growth & Sales (Pipeline)

- **Audience:** Employers, plans, systems, and operational stakeholders.
- **Goal:** Show measurable value and implementation practicality.

### Practices
- Lead with the stakeholder pain and operational objective.
- Use sourced claims from core context files.
- Frame comparisons respectfully and category-accurately.

---

## 3. Product & Technical

- **Audience:** Implementers, RevOps, analytics, and systems partners.
- **Goal:** Precision without unnecessary complexity.

Use explicit schema language for:
- MAP evidence extraction fields.
- Confidence-tier definitions.
- Follow-up action requirements.

---

## 4. Translation Layer (Quick Reference)

| Context | Emphasis |
| :--- | :--- |
| Website/landing | Customer-facing editorial voice |
| Ads/lifecycle | Growth voice + pillar alignment |
| Sales/partner docs | Operational value + proof |
| Internal runbooks | Technical precision + action ownership |

Do not mix patient-support copy with aggressive enterprise jargon in the same asset.
